# Tempo Host Instruction Book

**Formats:** Same content is available as **`HostInstructionBook.html`** (open in a browser → Print → Save as PDF) and **`Tempo-Host-Instruction-Book.pdf`** (generated for repo releases).

**For hosts running music bingo from scratch — including rounds, prep, save, and print.**

---

## 1. What you’re doing (big picture)

You’re running a **music bingo game**: songs play, players mark their cards, someone yells bingo.

Tempo helps you:

- Pull songs from **Spotify** and/or **YouTube Music**
- Build **rounds** (like chapters of your event)
- **Prep** and **save** rounds **before** the crowd arrives
- **Print paper cards** for people who play with markers (“daubers”)

Your screen has **tabs**. Think:

| Tab | Main job |
| --- | --- |
| **Manager** | Pick playlists, plan rounds, connect music accounts |
| **Game** | Run the live game (play snippets, see controls) |

---

## 2. Before opening Tempo

1. Use a **computer with Chrome or Edge** (safest bet).
2. Have **Spotify** and/or **YouTube Music** ready — whichever matches how you’ll host.
3. Know your **host login** for Tempo (whoever creates or owns the room).

---

## 3. First-time setup in the room

### Step A — Open your room

- Join your Tempo room as the **host** (only hosts control playlists and music).

### Step B — Connect playback

1. Open **Connection** (or Spotify / devices — wording depends on your build).
2. Connect **Spotify** if you’re playing from Spotify.
3. Connect **YouTube Music** if you’re playing from YouTube-only playlists.

**Why this matters:** Tempo can’t load your playlists until the platform knows it’s *you*.

### Step C — Pick where sound comes out

- Choose the **speaker device** (DJ booth, mixer, Bluetooth speaker, etc.).
- Test volume once — saves panic later.

---

## 4. Playlists and the “song pool”

### Two common setups

**Five playlists (5 × 15 mode)**  
- You pick **five playlists**.  
- Each playlist feeds **one column** on the big bingo grid (when the game is set up that way).  
- Great when each playlist has a **theme** (decades, genres, artists).

**One huge playlist (1 × 75 mode)**  
- One playlist with **lots of songs**.  
- Everything mixes into **one big pool**.  
- Great for simpler events.

Tempo will tell you if something doesn’t fit (too few songs, etc.). Fix playlists and try again.

---

## 5. Rounds — treat them like “chapters”

**Rounds** split your event:

- **Round 1** — opening games  
- **Round 2** — themed round  
- **Round 3** — finals  

Each round has:

- A **name**
- **Playlists** attached to that round  
- A **status** like planned → active → done  

You move rounds forward as your schedule moves — not because Tempo forces you.

---

## 6. Prep without embarrassing yourself live (“Load for prep”)

You don’t have to **start** the live game just to get organized.

**Load for prep** means:

- That round’s playlists move into the **mix**
- Your **Bingo pattern** controls sync to what that round needs  
- You can tweak things **before** players hear anything  

Use this when you’re alone setting up: load Round 2, check pattern, save, print cards — **no crowd needed**.

---

## 7. Save round — freezing your work

**Save round** stores a **snapshot** of that round’s setup at save time:

- Which tracks ended up in the locked pool (after rules trim duplicates / layout)
- Lets **Start Game** later follow a **predictable order** when you want it

Think of it as **bookmarking** the round so show night matches rehearsal.

**Tip:** After big edits (new playlists, new finalize), **save again** so the snapshot stays honest.

---

## 8. Finalize mix — the “lock it in” button

**Finalize** (sometimes labeled **Finalize mix**):

- Builds the **official bingo song pool** from what’s selected  
- Applies layout rules (like **5×15** trimming)  
- Lets digital cards, projector lists, and prints line up  

You’ll finalize when:

- You’re ready for **real cards / real PDFs**, or  
- You’re about to **start** play  

If Tempo says finalize failed: usually playlists didn’t load, Spotify/YouTube hiccuped, or you hit a rate limit. Refresh library, wait a minute, try again.

---

## 9. Printable PDF cards

There’s a **Cards per PDF** number (often **30** by default = 30 different sheets).

**Important:**  
**Round & event management → Print PDF** and **Game tab → Download PDF** use the **same engine** now:

- Same random layout rules  
- Same bingo geometry  
- Only difference: PDF **subtitle** may show the **round name** from the manager modal  

You **must finalize** before printing works — same rule everywhere.

**Where to set card count:** Manager tab *and* inside **Round & event management** — they stay in sync.

---

## 10. Round & event management (the popup)

Open **Round prep & PDF** from Manager.

Inside you’ll see:

- **Event overview** — how many rounds are planned / done  
- **Cards per PDF** — how many sheets  
- **Per-round buttons:** Load for prep, Save round, Print PDF, Start (when it applies)

Use this popup when you’re thinking **event-wide**, not just “right now on stage.”

---

## 11. Running the live multi-round game

### Typical flow

1. **Manager:** playlists assigned, rounds planned  
2. **Prep:** Load round → finalize if needed → save round → print PDF if you need paper  
3. **Game tab:** **Start Game** when the audience is ready  
4. Music plays in **snippets** — Tempo reveals titles on the **public display**  
5. Players mark squares → bingo gets checked  

### Moving to the next round

When a round ends:

- Mark it **complete** / advance according to your UI  
- **Load for prep** (or equivalent) for the **next** round’s playlists  
- Pattern + pool should match that round  
- Start music again when you’re ready  

### If something looks wrong on the projector

- **Call list columns** should match playlists only after **finalize** built the official columns  
- If you switched playlists mid-debug, **finalize again** so everyone sees the same pool  

---

## 12. Quick troubleshooting

| Problem | Try this |
| --- | --- |
| Empty playlists | Re-open Connection; refresh library |
| “Finalize failed” | Wait out Spotify limits; fewer playlist pulls at once |
| Printed cards look random-weird | Make sure you **finalized** after last playlist change |
| YouTube vs Spotify confusion | Connect the service you’re actually testing |
| Lost browser tabs | Rejoin room — cards often rebuild from server state |

---

## 13. Cheat-sheet checklist (night before)

- [ ] Login works  
- [ ] Spotify **or** YouTube connected  
- [ ] Speaker device picked  
- [ ] Each round has playlists  
- [ ] Load-for-prep tested once per round you care about  
- [ ] Save round after happy rehearsal  
- [ ] PDF printed if you need paper  
- [ ] Pattern + free-space checked  
- [ ] Public display URL/bookmark ready for projector  

---

## 14. You’ve got this

Hosting is mostly **slow prep + loud payoff**.

Use **Load for prep** and **Save round** so you’re not learning menus while a crowd stares at you.

Questions belong in your crew chat — or whoever shipped your Tempo build.

**Have fun running the show.**
